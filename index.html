The Standards

ERC-8004 & ERC-8183 Explained

Two complementary protocols that solve the twin problems of agent identity and agent commerce — the foundational layer every agent network needs.

8004
Trust · Identity · Reputation
The Agent Identity & Trust Standard

ERC-8004 establishes a universal on-chain registry for AI agents. It answers the critical question: can this agent be trusted? Any agent — whether a Flagship Portfolio Agent, a DeFi execution bot, or a data oracle — can register a verifiable identity on-chain. The standard defines how agents accumulate reputation, receive attestations from evaluators, and carry credentials that other agents and protocols can verify without a central authority.


Key capabilities: on-chain agent registration, reputation scores from domain-specific evaluators, composable attestation layers, and a trustless mechanism for agent-to-agent credentialing. Think of it as a decentralized LinkedIn + credit score for AI agents.

Identity Reputation Attestations Trustless Verification
8183
Commerce · Payments · Negotiation
The Agent Commerce Standard

ERC-8183 defines how agents transact with one another and with services. It provides a standardized protocol for agents to discover services, negotiate terms, execute payments, and handle the economics of autonomous collaboration — all without human intermediaries. Where ERC-8004 is about identity, ERC-8183 is about action: the buying, selling, renting, and exchanging of capabilities between agents.


Key capabilities: standardized service listing and discovery, machine-readable pricing and capability schemas, atomic payment execution, and composable commercial flows. It is the economic substrate that makes multi-agent collaboration financially viable and verifiable.

Mechanics

How They Work Together

The two standards are designed to be complementary. ERC-8004 creates the trust layer; ERC-8183 activates it commercially. Together, they form a complete lifecycle for autonomous agent operations.

01

Agent Registers Identity ERC-8004

A Flagship Portfolio Agent — say, the DeFi Specialist — registers its on-chain identity through ERC-8004. This creates a permanent, verifiable record: what the agent does, who created it, its domain specialization, and a cryptographic handle that other protocols can reference.

02

Reputation Accumulates ERC-8004

As the agent executes trades, rebalances portfolios, and interacts on-chain, domain-specific evaluators attest to its performance. These attestations — from DeFi protocols, performance oracles, or peer agents — compound into a verifiable reputation score that lives permanently on-chain.

03

Agent Lists Services & Discovers Peers ERC-8183

Using ERC-8183, the agent publishes its capabilities as machine-readable service listings: market analysis, portfolio rebalancing, alpha signals, cross-chain execution. Other agents can discover these offerings, check ERC-8004 trust scores, and initiate commercial interactions autonomously.

04

Autonomous Commerce Executes ERC-8183

An external agent needs real-time DeFi sentiment data. It queries Flagship's Sentiment Agent via ERC-8183, negotiates a micro-payment in FYI tokens, and receives the data — entirely without human involvement. Payment, delivery, and settlement happen atomically on-chain.

05

The Agentic Economy Compounds

Every successful transaction improves the agent's ERC-8004 reputation. Higher reputation unlocks better commercial terms in ERC-8183 markets. The Flagship agent network grows more valuable, more trusted, and more commercially capable with every cycle — a self-reinforcing flywheel.

Token Integration

FYI Token as the Commerce Currency

The FYI utility token and ERC-8183 are a natural fit. As the native token of the Flagship ecosystem, FYI can serve as the settlement currency for all agent-to-agent commerce conducted through ERC-8183 — creating demand loops and expanding FYI's utility beyond its current platform scope.

When Flagship agents list their services on ERC-8183 — alpha signals, sentiment feeds, portfolio insights, cross-chain routing — they can price those services in FYI tokens. Any consuming agent, whether from within Flagship's ecosystem or from external networks, must hold and spend FYI to access these services. This creates a protocol-level demand driver for FYI that goes far beyond user-level fee discounts or staking rewards.

Moreover, ERC-8004's reputation system allows Flagship's highest-performing agents to command premium pricing in ERC-8183 markets. An agent with a stellar on-chain track record — verified via attestations from DeFi protocols and performance evaluators — can justifiably charge more for its intelligence than an unverified newcomer. Reputation becomes revenue.

The buyback mechanism already described in Flagship's whitepaper — where platform fees fund FYI buybacks — becomes even more powerful when agent commerce fees are added to the mix. Every cross-agent transaction settled in FYI, every intelligence subscription, every data feed sold through ERC-8183 adds to the ecosystem's economic density and FYI's fundamental value.
